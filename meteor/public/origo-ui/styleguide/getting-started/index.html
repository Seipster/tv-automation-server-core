Origo Styleguide

Getting started

This article helps you start a new Origo-style site from scratch.

Download Origo design

The Origo design framework is available from NRK's Stash server. You have three options:

Download the zip-file

Download the Origo design zip archive.

Install from git with Bower

If your project is using Bower, you can install the Origo design from the Git SSH URL: 

bower install --save ssh://git@stash.nrk.no:7999/origo/origo-design.git#~1.1.1

Note that in this case, you need to specify which version you are installing. Available versions are all tags, see Stash for a full list.

Setting up the template

Here is the minimal page template you want to use for an Origo design-powered site: 

<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" href="/path/to/origo/dist/origo.css">
    <meta charset="utf-8">
    <meta http-equiv="x-ua-compatible" content="IE=edge">
  </head>
  <body>

  </body>
</html>

Loading the dist/origo.css file is your best option as it is a single file containing all the rules nicely minified. If you are manually copying files around, you need to make sure that path/to/origo contains the dist, fonts and images directories.

Using SVG icons

If you also want to use the SVG icons, you also need to make the scripts directory available. Additionally, your template needs to include the icon container and load the icons script: 

<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" href="/path/to/origo/dist/origo.css">
    <meta charset="utf-8">
    <meta http-equiv="x-ua-compatible" content="IE=edge">
  </head>
  <body>
    <div id="icon-container" class="hidden"></div>
    <script src="/path/to/origo/scripts/icons.js"></script>
  </body>
</html>

This will basically load all the SVG icons onto the page, making it possible to refer to them by id later in the document, as described in the styleguide.

Creating a masthead

A very common feature is the dark masthead on top of the page. How can Origo design be used to create something like it? As with anything that involves layout, you start with the grid layout system. It works just like tables: 

<div class="row dark">
  <div class="col c4">
    <div class="media mod">
      <img class="media-elem mrs" src="../../images/nrk.svg" width="58">
      <div class="bd mls">My new Origo site</div>
    </div>
  </div>
  <div class="col right">
    <p>Some tools all the way to the right</p>
  </div>
</div>

The dark class enables the dark theme. The mod class gives the div block level spacing semantics. The media, media-elem, and bd classes describe a media element. row, col, and c4 all belong to the grid layout system. The result is this:

Finding the right typography

If you are working from Photoshop design sketches, you probably need to recreate specific typography. The styleguide contains a typographic glossary which can be used for this purpose. Find the right set of typography rules on the right, and put them to use with the most appropriate class names to the left. For instance, if some part of your interface calls for the button typography but without the button borders and other styles, you can use the text-s-caps class: 

<a href="#" class="text-s-caps">Click it, I dare ya</a>

If you need to tone it down, you can add the subtle class:
