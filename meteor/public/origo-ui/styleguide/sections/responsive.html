Responsive design

The approach to responsive design is "mobile first", or said differently: "small first". So default styles are intended for the smallest screens, then media queries are used to override these styles for bigger screens. There are currently five sizes: the default, small (> 640px wide screens) , medium (> 992px wide screens), large (> 1200px wide screens) and xlarge (> 1800px wide screens).

Some base styles vary between the various screen sizes - for example, h1 headings are smaller on the smallest screens.

Responsive layout

The layout can be individually specified for each of the three screen sizes. The default column-width classes (e.g. c3 and so on) apply to the smallest screens, and can be overridden for medium and large screens. To do so, use classes prefixed with rm- for medium screens, and rl- for large screens. When the 12 column grid is not sufficient to lay out a component, there are additional classes to set the width of columns: s1-5, which gives you a column one fifth (e.g. 20%) of the available width, and s1-7. They are also available with rm- and rl- prefixes, for responsive applications.
