Material design input fields

Input fields can be implemented as simple inputs, without any effects

 

Alternatively, a little bit more crufty markup will give you some subtle material design-inspired visual effects.

For input elements that can be individually saved or applied, you can add an action button which will be visible while editing.

Fields can have two actions associated with them: one that is available while editing the field (edit-action), and one that is available when the field does not have focus (view-action).

Input fields can have action buttons on the left as well. These are always visible.

Input fields that have been recently changed are highlighted. Add the input-hl class to the mdi to achieve this effect.

Sometimes users enter undesired values into our inputs. Flag the mdi container element of those inputs with the error class. The error message element is optional.
